# SkillCompass — сценарий защиты (5–7 минут)

Речь для презентации DataPulse. Запустите API и дашборд заранее (см. `USER_STEPS.md`, шаг 5).

---

## Вступление (30 сек)

> **SkillCompass** — data-driven сервис для IT-аналитиков. Мы собираем вакансии с **HH.ru** и **SuperJob**, анализируем рынок по **четырём ролям** и даём **ML-прогноз зарплаты**.
>
> Ниша узкая: бизнес-, системный, продуктовый аналитик и аналитик данных — без «кредитных аналитиков» и прочего шума.

**Цифры:** ~699 вакансий · HH 615 + SuperJob 84 · 2 источника · CatBoost-регрессия.

---

## 1. Overview — рынок в цифрах (1–2 мин)

**URL:** http://localhost:8501 → **Overview**

Что показать:

- Общее число вакансий и разбивка **по 4 ролям**
- Сравнение зарплат: например, системный vs бизнес-аналитик
- Источники: HH и SuperJob

**Что сказать:**

> Мы собираем данные отдельным запросом на каждую роль, дедуплицируем по `(source, id)`. Фильтр отсекает нерелевантные вакансии. Видно, что системных и бизнес-аналитиков на рынке примерно поровну, продуктовых и data — чуть меньше.

---

## 2. Predictions — ML-прогноз (1–2 мин)

**Страница:** **Predictions**

Действия на экране:

1. Роль: **Системный аналитик**
2. Опыт: **Опыт 1-3 года**
3. Город: **Москва**
4. Источник: **hh**
5. Нажать **«Рассчитать»**

**Что сказать:**

> Модель — **CatBoost**, регрессия зарплаты в рублях в месяц. Признаки: роль, опыт, регион, источник, тип занятости, наличие диапазона в похожих вакансиях. На тесте MAE около **53 000 ₽**, модель **лучше naive baseline** (средняя по роли).
>
> Это не «точная зарплата», а ориентир по рынку для профиля кандидата.

Покажите блок **метрик** (MAE, RMSE, beats_naive).

---

## 3. Data — прозрачность данных (1 мин)

**Страница:** **Data**

Действия:

- Фильтр: **продуктовый аналитик** (или любая роль)
- Показать таблицу: название, работодатель, зарплата, ссылка

**Что сказать:**

> Все данные можно проверить в таблице. API отдаёт те же записи с пагинацией — это основа для дашборда и внешних интеграций.

---

## 4. Monitoring — инфраструктура (30 сек)

**Страница:** **Monitoring**

**Что сказать:**

> FastAPI-сервис отдаёт health, stats и predict. Дашборд на Streamlit ходит в API. Для прод-сценария — **Docker Compose**: PostgreSQL, ingestor по расписанию (каждые 24 ч), API и dashboard.

---

## 5. Swagger (опционально, 30 сек)

**URL:** http://127.0.0.1:8000/docs

- `GET /data/stats` — Execute
- `POST /predict` — тело как в калькуляторе:

```json
{
  "analyst_role": "systems_analyst",
  "experience": "Опыт 1-3 года",
  "area": "Москва",
  "source": "hh",
  "employment": "unknown",
  "has_salary_range": 0
}
```

## Архитектура (если спросят)

```
HH.ru + SuperJob  →  Ingestor  →  PostgreSQL / Parquet
                                        ↓

                                        ↓
                              FastAPI  ←→  Streamlit
```

**Стек:** Python, pandas, CatBoost, FastAPI, Streamlit, Docker, PostgreSQL.

## Типичные вопросы

| Вопрос | Ответ |
|--------|-------|
| Почему только 4 роли? | Узкая ниша IT-аналитиков — чище данные и понятнее продукт |
| Откуда данные? | HH (API + HTML fallback при 403) и SuperJob REST API |
| Почему MAE 53k? | Зарплаты сильно разбросаны; 196 вакансий с указанной зарплатой из 699 |
| Зачем Docker? | Требование курса: контейнеры, БД, scheduled ingest |
| Как обновить данные? | `fetch_all.py` + `train_salary.py` или ingestor в Docker |
