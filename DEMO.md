## Вступление (30 сек)

> **SkillCompass** — data-driven сервис для IT-аналитиков. Собираем вакансии с **HH.ru** и **SuperJob**, анализируем рынок по **четырём ролям**, даём **ML-прогноз зарплаты** и **топ навыков**.
>
> Ниша узкая: BA, SA, PA, DA — без «кредитных аналитиков» и прочего шума.

**Цифры:** ~**3075** вакансий · HH ~2990 + SuperJob ~87 · CatBoost-регрессия · MAE ~**44k** ₽.

---

## 1. Общая статистика (1–2 мин)

**Меню:** **Общая статистика**

Показать:

- KPI: всего вакансий, с зарплатой, источники
- Stacked bar: вакансии по роли (HH + SuperJob)
- Средняя ЗП по ролям
- График динамики зарплат (если есть snapshots)

**Что сказать:**

> Сбор — отдельные запросы RU+EN на каждую роль, merge потоков, dedup по `(source, id)`, фильтр шума, cap зарплат 30k–600k. Роль определяем из **названия** вакансии.

---

## 2. Прогноз зарплаты и навыков (1–2 мин)

**Меню:** **Прогноз зарплаты и навыков**

1. Роль: **Системный аналитик**
2. Опыт: **Опыт 1-3 года**
3. Город: **Москва**
4. **Рассчитать**

**Что сказать:**

> CatBoost-регрессия. Признаки: роль, опыт, город, источник, формат указания ЗП (`salary_spec`). MAE ~44k на тесте, модель **лучше naive baseline**. Ниже — топ навыков HH для выбранного профиля (не входят в модель ЗП).

## 3. Данные (1 мин)

**Меню:** **Данные**

- Фильтр: продуктовый аналитик

## 4. Мониторинг (30 сек)

**Меню:** **Мониторинг**

> FastAPI + PostgreSQL + Streamlit. Docker Compose на сервере, ingestor по расписанию (4 ч). CI: push в main → auto-deploy.

## Типичные вопросы

| Вопрос | Ответ |
|--------|-------|
| Почему 4 роли? | Узкая ниша IT-аналитиков — чище данные |
| Откуда данные? | HH (API + HTML при 403), SuperJob REST |
| Почему не все с ЗП? | HH часто не показывает ₽ в списке; ~36% с зарплатой после enrich |
| Merge данных? | Два потока HH объединяются в `merge_and_clean_rows` |
| Валидация? | Stop-list, cap ЗП, Pydantic на API |
| Автотесты / Alembic? | Нет; схема через SQLAlchemy `create_all` |
| MAE ~44k? | Разброс зарплат большой; модель лучше «одной медианы» |
